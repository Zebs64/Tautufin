Import d'historique

Migre un historique de lecture existant vers la base interne. Trois formats acceptés (détection automatique) :

  • la base du plugin Jellyfin Playback Reporting (playback_reporting.db) ;
  • un backup de ce plugin (PlaybackReportingBackup-AAAAMMJJ-HHMMSS.tsv) ;
  • un backup Streamystats (fichier .json « Backup »).

L'import est idempotent : relancer ne crée pas de doublons (clé de déduplication : date + utilisateur + média), y compris entre les différents formats.

1. Choisir la source

Base playback_reporting.db, backup PlaybackReportingBackup-*.tsv ou backup Streamystats *.json (détection automatique).

2. Analyser puis importer

Les filtres de durée minimale actuels s'appliqueront : minimum_duration = {{ minimum_duration }}s, minimum_percent = {{ minimum_percent }}%.

Inférer l'historique depuis Jellyfin

Pour les visionnages jamais capturés en direct (avant la mise en service de Tautufin, ou pendant un arrêt), Jellyfin sait quels médias chaque utilisateur a marqués comme lus. On reconstitue une session « inférée » par média lu : date = dernière lecture connue, durée = durée totale du média (hypothèse 100 % vu). Le client, le transcodage et les visionnages multiples ne sont pas récupérables.

Sans danger : ces sessions sont marquées inférées et supprimables d'un clic. Relancer ne crée pas de doublon (les inférées du périmètre sont régénérées) et n'écrase jamais une lecture réelle déjà enregistrée à la même date.
